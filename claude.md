# IdleEmpire — 遊戲設計文件 (Claude 長期記憶)

> 這份文件是 Pai 與 Claude 共同打造的放置型像素遊戲的核心設計脈絡。
> 每次接手這個專案時，請 **先讀這份檔案**，再進行任何修改。
> 我（Claude）的角色：專業遊戲設計師，全權負責設計與實作；Pai 是督導與試玩者，負責回饋與方向把關。

---

## 1. 專案核心理念

打造一款 **「星露谷物語質感 × Northguard 經營策略」** 的放置型網頁遊戲，玩家透過單純點擊分配資源，享受觀察 NPC 各司其職、團隊與聚落不斷擴張的「家近經營感」。

**核心情緒目標**：
- 看著小人忙進忙出的療癒感
- 資源從 0 → 1 → 100 的成長感
- 每一個建築、每一隻 NPC 都「活著」的世界感
- 不催促、不焦慮，玩家可以隨時離開又回來

**反設計原則（避免做的事）**：
- 不要繁雜操作（沒有戰鬥微操、沒有複雜選單套娃）
- 不要逼玩家盯著螢幕（放下手機回來資源還在動）
- 不要假像素（程式化生成的圖、AI 生圖反鋸齒、低密度色塊）
- 不要任何形式的「等待付費跳過」誘導

## 2. 遊戲視角與技術架構

| 項目 | 規格 |
| --- | --- |
| 視角 | 2D 俯視角（top-down），偏 3/4 透視（建築有高度感） |
| 平台 | 單檔 HTML（HTML5 Canvas + 原生 JS），不依賴框架 |
| 美術風格 | 16x16 / 32x32 像素風，星露谷色票（柔和飽和、暖色調） |
| 解析度 | 視窗自適應，內部用 pixel-perfect scaling，禁止反鋸齒 |
| 存檔 | localStorage 自動存（每 30 秒 + 關閉前） |
| 資產來源 | 真人手繪免費像素包（Cup Nooble Sprout Lands / Kenney / LimeZu），CC0 / 免費商用 |

## 3. 資源系統

| 資源 | 來源 | 用途 | 圖示色調 |
| --- | --- | --- | --- |
| 金幣 (Gold) | 市場販售 / 戰利品 | 招募 NPC、買藥水、升級建築 | 金黃 |
| 木材 (Wood) | 伐木場 + 採集師 | 建造與升級建築 | 棕色 |
| 礦石 (Ore) | 礦坑 + 採礦師 | 強化軍人裝備、特殊建築 | 灰青 |
| 糧食 (Food) | 農舍（農夫種稻）+ 採集師（野果） | 餵食 NPC，飢餓度恢復 | 麥黃 |
| 藥水 (Potion) | 藥水房（藥水師） | 恢復 NPC 血量 | 翠綠 |

**資源平衡核心法則**：
- 早期糧食是瓶頸（NPC 越多吃越快）
- 中期木材是瓶頸（建築擴張）
- 後期金幣是瓶頸（招募高階兵種、特殊建築）

## 4. NPC 系統

### 4.1 職業（v1）

| 職業 | 工作行為 | 產出 |
| --- | --- | --- |
| 農夫 (Farmer) | 巡迴種植/收成稻田 | 糧食 |
| 採集師 (Gatherer) | 在森林採野果、撿樹枝 | 糧食 + 少量木材 |
| 採礦師 (Miner) | 到山區礦點挖礦 | 礦石 |
| 藥水師 (Alchemist) | 在藥水房調配 | 藥水 |

### 4.2 屬性

- **HP（血量）**：被怪物攻擊或飢餓歸零後扣血
- **Hunger（飢餓度）**：100 → 0，工作時下降，吃糧食恢復
- **死亡邏輯**：HP 歸零 → 倒地動畫 → 從世界移除 → 必須重新招募
- **沒有復活**（這是設計選擇，讓玩家對 NPC 有羈絆與責任感）

### 4.3 行為狀態機

```
IDLE → SEEK_WORK → MOVE_TO_WORKSITE → WORK → MOVE_HOME → DEPOSIT → EAT_IF_HUNGRY → SLEEP_IF_NIGHT
                                            ↓
                                          DEAD（HP=0 或 Hunger=0 持續一段時間）
```

### 4.4 動畫規格（嚴格遵守）

每個 NPC 需要 **4 方向 × 多動作**：
- 方向：上 / 下 / 左 / 右（左右共用 sprite 翻轉即可）
- 動作：idle（呼吸）、walk（走路 4-6 幀）、work（揮鋤/採集 4 幀）、attack（戰鬥單位）、hurt（受傷）、die（倒下）

## 5. 建築系統

### 5.1 v1 建築清單

| 建築 | 功能 | 容納 NPC | 解鎖條件 |
| --- | --- | --- | --- |
| 主城 (Town Hall) | 起始建築，玩家家 | — | 預設 |
| 農舍 (Farm) | 雇用農夫，種稻 | 1-3 | 木 30 |
| 伐木場 (Lumber Yard) | 自動產木 + 雇用採集師 | 1-2 | 木 50 |
| 礦坑 (Mine) | 雇用採礦師 | 1-2 | 木 80 + 金 50 |
| 藥水房 (Alchemy Lab) | 雇用藥水師 | 1 | 木 60 + 礦 20 |
| 市場 (Market) | 販售資源換金幣 | — | 木 40 |
| 兵營 (Barracks)（後續） | 雇用軍人 | 多 | — |

### 5.2 建造機制
- 點擊 UI 建造按鈕 → 進入「擺放模式」→ 半透明預覽跟隨滑鼠
- 合法格子綠色高亮，違法（重疊/水域）紅色高亮
- 確認 → 扣資源 → 蓋好的建築有「正在建造」的鷹架動畫，數秒後完工

## 6. 世界與地圖

- 開放世界 tile map，遠大於螢幕，可滑動 + 縮放鏡頭
- 地形類型：草地、森林、河流、山脈、沙地（不同生物群系決定資源點）
- 資源點散佈：野生果樹（採集師）、礦脈（採礦師）、稻田位（農舍可種植區）
- 晝夜循環：實時 5 分鐘 = 遊戲一日，夜晚有遮罩、NPC 回家睡覺
- 後續：怪物會在夜晚從地圖邊緣出現

---

## 7. 玩家操作模式（極簡）

只有 **3 種互動**：
1. **滑動鏡頭**：拖曳 / WASD / 邊緣滾動
2. **點擊**：點建築看面板、點 NPC 看狀態、點空地建造
3. **快捷列**：建造選單、市場、設定

**沒有的東西**：沒有 RTS 框選、沒有戰鬥微操、沒有任務面板套娃、沒有快捷鍵記憶負擔。

## 8. 美術品質規範（不可妥協）

- ✅ **真人手繪像素素材**（Cup Nooble Sprout Lands、Kenney、LimeZu 等）
- ✅ 透明通道乾淨，無描邊雜訊、無白底、無色暈
- ✅ 風格統一：所有素材像素密度一致、調色盤一致
- ❌ **絕對禁止**程式化繪製像素人（rect 拼角色那種沒質感的做法）
- ❌ **絕對禁止** AI 生圖（反鋸齒、偽透明、邊緣有色暈）

## 9. 音樂音效

- 背景音樂：和諧輕快的 chiptune（CC0 / 免費商用）
- 音效：點擊、建造、收成、警告、招募成功

## 11. Pai 與 Claude 的合作守則

- **Pai 主導方向**：他會給回饋、提建議、改變方向，Claude 必須認真聽
- **Claude 主導執行**：技術選型、實作細節、資源平衡數值，Claude 自己決定，Pai 不需要被技術細節煩擾
- **誠實 > 討好**：Claude 發現問題（例如某個玩法可能不好玩、某個美術風格不一致）必須直接告訴 Pai，不要為了討好而隱瞞
- **遇到限制要說**：例如「我不能直接呼叫 Gemini」這種限制必須立刻坦白，並提出更好的替代方案
- **每個版本都要可玩**：絕不交付半成品、絕不留假按鈕，每次端出去玩家都要能完整體驗一個循環

## 14. v1 已知限制與下一階段路線

**v1 限制**（誠實揭露）：
- 所有 NPC 共用 `farmer_base` sprite + 職業色光暈區分（職業視覺差異不夠強烈）
- 簡易直線移動，沒有 A* 尋路（NPC 會穿越水域看起來怪）
- 建築物從 House.png 大圖切片，每種建築其實是同一張圖的不同區塊（v2 要找專屬 sprite）
- 沒有戰鬥、怪物、防禦塔（v2）
- 沒有作物田地視覺（農夫直接在農舍旁工作）

**v2 優先項目**：
1. **A* 尋路 + 水域避障**（基礎品質飛躍）
2. **農舍周圍的稻田 tile 動態渲染**（用 soil/ + corn/ 4 階段生長）
3. **每種職業換不同 base sprite**（找 LimeZu 或 LPC 額外角色）
4. **怪物 + 戰鬥**（兵營、劍士 NPC、夜晚 spawn 怪）
5. **建築升級**（Lv1→Lv2→Lv3，產量 / 容量 / 速度三選一）
